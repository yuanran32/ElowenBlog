# 博客改造建议

这份文档用于记录后续可以继续优化的方向。建议先做收益高、风险低的改动，再逐步处理视觉和内容结构。

## 优先级一：提升技术博客阅读体验

### 1. 文章页增加目录 TOC

当前文章页只有正文内容，文章变长后不方便快速定位章节。建议在文章详情页自动提取 `h2`、`h3` 标题，生成文章目录。

可以考虑两种形式：

- 正文顶部显示目录，移动端友好，实现简单。
- 桌面端右侧固定目录，长文阅读体验更好。

推荐先实现正文顶部目录，后续再增强为响应式侧边目录。

### 2. 强化代码块样式

技术博客会频繁展示代码，建议优化代码块可读性。

已完成：

- 更清晰的代码块背景和边框。
- 行内代码样式。
- 复制按钮。

后续还可以增加：

- 可选的文件名展示。

这部分能明显提升文章的专业感。

### 3. 日期、标签、更新信息统一展示

文章列表和文章详情页已经统一展示：

- 发布时间。
- 更新时间。
- 标签。
- 阅读时间。

后续可以继续优化信息密度和移动端排版。

## 优先级二：完善内容模型

### 1. 文章增加草稿字段

建议给博客文章 frontmatter 增加：

```yaml
draft: true
```

构建时过滤草稿文章，这样本地可以放心写未完成内容，不会误发布。

### 2. 文章增加系列字段

已给文章增加：

```yaml
series: 前端测试
```

适合整理专题内容，例如：

- 前端测试。
- Web 工程化。
- Astro 博客搭建。
- 数据可视化。

已增加系列页，把同一主题文章聚合展示。

### 3. 项目增加技术栈和展示链接

当前项目内容偏简单，建议给项目 frontmatter 增加：

```yaml
techStack:
  - Astro
  - Tailwind CSS
  - TypeScript
demoUrl: https://example.com
githubUrl: https://github.com/yuanran32/myBlog
role: 独立开发
highlights:
  - 支持 Markdown 内容管理
  - 配置 RSS、站点地图和基础 SEO
```

这样项目页会更像作品集，而不是普通链接列表。

## 优先级三：首页结构优化

当前首页已经能展示个人介绍、精选项目和近期文章，但整体还比较接近模板。建议改成更明确的个人技术站结构。

推荐首页结构：

1. 顶部个人定位。
2. 当前关注方向。
3. 近期文章。
4. 精选项目。
5. 联系入口。

其中「当前关注方向」可以写成：

- 前端开发。
- Web 工程化。
- 前端测试。
- 全栈应用。
- 数据可视化。

首页不需要放太多说明文字，重点是让访问者能快速进入文章和项目。

## 优先级四：简历页改造

当前简历页是 Markdown 长文，维护方便，但展示效果一般。后续可以把 `/resume` 做成更结构化的页面。

建议拆成以下模块：

- 基本信息。
- 技术栈。
- 项目经历。
- 实习 / 工作经历。
- 教育经历。
- 联系方式。

项目经历建议突出：

- 项目背景。
- 技术方案。
- 个人职责。
- 难点与解决方案。
- 可量化结果。

如果简历主要用于求职，建议优先优化项目经历，而不是堆技能关键词。

## 优先级五：SEO 和分享优化

当前已经有基础 SEO、RSS 和 sitemap。后续可以继续补充：

- 默认 Open Graph 分享图。已完成。
- 每篇文章单独的 `seo.image`。
- 更完整的 `description`。
- 首页和文章页的结构化数据。已完成。

这部分对搜索和社交分享有帮助，但优先级低于文章阅读体验和内容模型。

## 已完成的工程增强

- 开发环境显示 `draft: true` 草稿，生产构建自动隐藏草稿。
- RSS 增加标签、系列和更稳妥的摘要兜底。
- 首页增加 `WebSite` / `Person` JSON-LD。
- 文章页增加 `BlogPosting` JSON-LD。

## 推荐下一步

建议下一轮优先做这三项：

1. 文章页增加目录 TOC。
2. 项目增加 `techStack`、`demoUrl`、`highlights` 字段。
3. 博客文章增加 `draft` 字段并在构建时过滤草稿。

这三项对技术博客的长期维护最有价值，改动范围也比较可控。
